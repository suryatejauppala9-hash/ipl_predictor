# IPL Intelligence 2026

A full-stack cricket analytics application that simulates, predicts, and visualises IPL matches using machine learning trained on 17 years of ball-by-ball data.

## What it does

IPL Intelligence gives you two ML-powered engines under one interface:

**Win Predictor** — An XGBoost classifier trained on 13 match-level features (rolling strike rates, economy rates, batting averages, home advantage, toss outcome, head-to-head win %) to predict which team wins before the first ball is bowled. Supports pre-match mode where toss is unknown — the model averages predictions across all four possible toss scenarios and breaks them down for you.

**Match Simulator** — Runs 6,767 full ball-by-ball T20 simulations using a second XGBoost model trained on cumulative batsman/bowler stats per delivery. Produces score distributions, P10/P90 ranges, a ball outcome breakdown (dots, 1s, 2s, fours, sixes, wickets), full scorecards, and a live win-probability momentum graph that updates after every over.

Everything else — Ideal XI generation, the custom XI builder, squad viewer, and the Impact Player designator — feeds into or out of these two engines.

## Dataset

**[IPL Dataset 2008–2025](https://www.kaggle.com/datasets/chaitu20/ipl-dataset2008-2025)** by chaitu20 on Kaggle.

Ball-by-ball data covering every IPL match from the inaugural 2008 season through 2025 — over a decade and a half of cricket reduced to individual deliveries. The raw CSV (`ipl_data.csv`) contains batter, bowler, runs, wicket, venue, toss, and match outcome columns for every ball of every game across all 10 active franchises.

> Download the dataset from Kaggle and place `ipl_data.csv` in the project root before running `data_cleaning.py`.

## Quickstart

### 1. Clone and install

```bash
git clone https://github.com/your-username/ipl-intelligence.git
cd ipl-intelligence
pip install -r requirements.txt
```

### 2. Get the data

Download `ipl_data.csv` from [Kaggle](https://www.kaggle.com/datasets/chaitu20/ipl-dataset2008-2025) and drop it in the project root.

### 3. Run the data pipeline

```bash
python data_cleaning.py
```

This generates six CSV files that the app depends on. You only need to run this once (or again if you update the raw data). It takes about 30–60 seconds depending on your machine.

What it does under the hood:
- Standardises team names across franchise renames (e.g. Delhi Daredevils → Delhi Capitals, Kings XI Punjab → Punjab Kings)
- Filters to the 10 currently active franchises
- Computes rolling last-5-match team form (strike rate, batting average, economy, bowling average)
- Builds per-player career batting and bowling stats
- Builds a batsman-vs-bowler matchup table (~every unique pairing in IPL history)
- Creates a ball-level feature dataset for the ball-outcome XGBoost model
- Outputs head-to-head win percentages for every franchise matchup

### 4. Start the server

```bash
python main.py
```

Then open **http://127.0.0.1:8000** in your browser.

## How the models work

### Match winner model

Features used at prediction time:

| Feature | Description |
|---|---|
| `team1_sr` / `team2_sr` | Rolling 5-match batting strike rate |
| `team1_bat_avg` / `team2_bat_avg` | Rolling 5-match batting average |
| `team1_econ` / `team2_econ` | Rolling 5-match bowling economy |
| `team1_bowl_avg` / `team2_bowl_avg` | Rolling 5-match bowling average |
| `team1_is_home` / `team2_is_home` | Home ground flag |
| `toss_winner_is_team1` | Toss outcome (or averaged across 4 scenarios) |
| `toss_decision_bat` | Bat/bowl decision |
| `team1_h2h_win_pct` | Historical head-to-head win % |

Model: `XGBClassifier` with 200 estimators, learning rate 0.05, max depth 4.

### Ball outcome model

Trained on every delivery in the dataset. Features are cumulative batter and bowler stats up to (but not including) the current ball — the model never looks ahead. Phase one-hot encoding (powerplay / middle / death) is included.

Model: `XGBClassifier` with 150 estimators, learning rate 0.08, max depth 5. Output classes: `{0, 1, 2, 3, 4, 6}` (5-run outcomes are mapped to 4).

Dismissal probability is modelled separately as a per-ball Bernoulli draw using the bowler's historical wicket rate, scaled by phase. Both models run independently and are combined during simulation.

### Simulation engine

Each of the 6,767 simulated matches:
1. Determines batting order based on toss (known or randomly drawn)
2. Simulates every ball across both 20-over innings using the ball outcome model (or calibrated heuristics if the model is unavailable)
3. Applies IPL-calibrated phase run-rates: powerplay 0.150 rpb, middle 0.148 rpb, death 0.195 rpb
4. Generates a ball log, over-by-over win probability, and a full scorecard
5. One representative match is returned for the UI — the rest contribute to the score distribution

Score calibration targets IPL 2024/25 averages (~191 runs per team).

## Features

### Win Prediction
- XGBoost prediction with confidence %
- Probability bar split between both teams
- Toss-unknown mode: 4-scenario breakdown with per-scenario win %
- Head-to-head, strike rate, batting average, and economy comparison

### Monte Carlo Simulation
- 6,767 full match simulations
- Mean score, standard deviation, P10 and P90 range per team
- Ball outcome distribution (dot%, 1s, fours, sixes, wicket%)
- Ball-by-ball log with phase labels and hover tooltips
- Full scorecard (batting + bowling)
- Over-by-over win probability momentum graph

### Momentum Graph
- Dual-colour area fill — team colour above 50%, opposing team colour below
- Wicket events highlighted with a red dot and glow
- Phase overlays: Powerplay, Middle, Death for both innings
- Smooth SVG path draw animation on load
- Interactive hover: over number, exact win probability, event detail, score

### Ideal XI Generator
- Three strategies: Balanced, Aggressive, Bowling-heavy
- Position-aware selection: openers ranked by SR, middle order by average, specialist bowlers last
- T20 rule enforcement: 5 bowlers, maximum 4 overs each
- Impact Player recommendation per team

### Custom XI Builder
- Full squad listed with career SR and bowler eligibility marked
- Click to select up to 11 players; selection order sets batting position
- Last 5 selected eligible bowlers bowl (T20: 5 bowlers)
- Feed directly into the simulation engine

### Squad Viewer
- Full 2026 squad rosters for all 10 franchises
- Filter by Batters / Bowlers / All-rounders
- Sort by any stat column
- Colour-coded stats: green (elite), white (average), red (poor)

### Impact Player
- Per team, per match — the player most likely to change the game
- Shows the bench player they would replace
- Follows the official IPL Impact Player rule: 1 substitution allowed after the toss, the replaced player cannot bowl or bat in the remaining innings

## API endpoints

The FastAPI backend exposes these endpoints directly if you want to build on top of it:

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Serves the UI |
| `POST` | `/predict` | Win prediction |
| `POST` | `/simulate` | Monte Carlo simulation |
| `POST` | `/simulate-custom` | Simulation with a custom XI |
| `GET` | `/ideal-xi/{team}` | Ideal XI for a team (`?style=balanced\|aggressive\|bowling`) |
| `GET` | `/playing11/{team}` | Default playing XI |
| `GET` | `/squad/{team}` | Full squad with stats |
| `GET` | `/player-stats` | All player stats (optional `?team=` filter) |

## Limitations & honest caveats

- **Player form is career-based, not in-season rolling.** The player stat tables use full IPL career aggregates. The team-level features use rolling 5-match form, but individual player selection does not yet incorporate current-season form.
- **Auction and trade moves after the 2026 retention deadline are not reflected.** Squads in `ipl_squads.py` are hard-coded from public information at the time of writing.
- **Weather, pitch conditions, and crowd effects are not modelled.** The venue advantage is a binary home/away flag, not a pitch report.
- **The simulation is not a full agent-based model.** Batting partnerships, running between wickets, and tactical field placements are abstracted.
- **Win probabilities are model outputs, not betting odds.** They reflect patterns in 17 years of data but should not be treated as ground truth for any specific match.
